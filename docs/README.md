# 文档中心

这份公开仓库的职责，是把 DeepDcc 讲清楚，而不是承载主开发源码。

如果你第一次接触 DeepDcc，建议按这个顺序阅读：

1. [产品总览](product-overview.md)
2. [功能详解](features.md)
3. [工作流程](workflow.md)
4. [示例与适用场景](examples.md)
5. [常见问题](faq.md)
6. [更新日志与版本入口](changelog-links.md)

## 你可以从这里得到什么

- DeepDcc 的定位和它与常见 AI 写作器的差异
- 核心能力和能力边界
- 一条从输入任务到正式导出的完整工作流
- 哪些场景更适合用 DeepDcc
- 当前测试版、更新入口与反馈方式

## 公开仓库不包含什么

- 主开发源码
- 内部测试与发布脚本
- 内部修复记录与设计审查材料
- 正式版收费链实现细节

公开真源仍然以官网为准：<https://deepdcc-pro.cn>
