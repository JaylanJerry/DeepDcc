# 产品总览

## 一句话理解

DeepDcc 是一套面向正式文档交付的 AI 工作台。它不是只会回答问题的聊天写作器，而是试图把“任务理解、结构规划、正文生成、预览核对、正式导出”收成一条连续工作流。

## 为什么会有 DeepDcc

很多 AI 写作工具的问题不在于“生成不出来”，而在于“交不出去”。

常见断裂包括：

- 结构和正文脱节
- 长文生成失控
- 公式只能作为源码或截图存在
- 预览和导出结果不一致
- 真正交稿前还要花大量时间手工返工

DeepDcc 的目标，是缩短这段返工链。

## 它的工作哲学

### 1. 先收口，再展开

不是一上来就整篇释放正文，而是先确认蓝图，先把结构这件事收住。

### 2. 面向交付，而不是只面向回答

最终关注点不是“回答像不像”，而是“这份文档能不能继续修改、预览、导出并交付”。

### 3. 让失败显式可见

当关键链路失败时，更偏向中止和提示，而不是继续伪造“看起来成功”的结果。

### 4. 尽量让内容状态统一

正文、预览、导出尽量围绕同一份稳定正文源运转，减少所见非所得。
